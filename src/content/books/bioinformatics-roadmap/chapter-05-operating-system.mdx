操作系统是你和硬件之间的管理者。没有操作系统，普通用户很难直接控制 CPU、内存、硬盘、显示器和网络设备。Windows、macOS、Linux 都是操作系统。它们看起来界面不同，但核心任务类似：管理资源，运行程序，组织文件，处理输入输出，并尽量让多个任务同时工作。

当你双击一个软件时，系统会把程序从硬盘加载到内存，让 CPU 执行其中的指令。正在运行的程序叫进程。一个浏览器可能有多个进程，一个代码编辑器也可能启动后台进程。任务管理器或活动监视器显示的 CPU 占用、内存占用，就是系统观察这些进程消耗资源的方式。

内存和硬盘容易混淆。硬盘负责长期保存，关机后文件还在；内存负责临时工作，速度快但断电后清空。打开大型软件、训练模型、处理大表格时，内存不足会导致明显卡顿甚至崩溃。学习编程后，你会经常遇到“这个程序为什么占这么多内存”的问题，这时就要理解数据结构、缓存和并发任务如何消耗资源。

操作系统还负责权限。为什么有些文件不能删除？为什么安装软件需要管理员权限？为什么脚本访问某些目录会失败？这是因为系统需要防止程序随意破坏环境。权限不是麻烦，而是安全边界。初学者不要习惯性地用最高权限运行所有东西，除非你清楚命令会做什么。

另一个重要概念是环境变量。环境变量可以理解成系统提供给程序的一组公共信息，例如 `PATH` 告诉系统到哪些目录寻找可执行程序。当你在命令行输入 `python`，系统能找到 Python，通常就是因为 Python 所在目录被加入了 `PATH`。很多安装问题，本质都是环境变量没有配置好。

本章练习：打开任务管理器，观察浏览器、编辑器和系统进程的 CPU、内存占用。然后在命令行输入查看 Python 版本的命令，例如 `python --version` 或 `py --version`。如果系统提示找不到命令，记录下来，不要急着乱装软件，先思考：系统为什么不知道去哪里找它？

常见误区是把“电脑卡”当成单一问题。实际上可能是 CPU 忙、内存不足、硬盘读写慢、网络等待、后台进程太多，或者某个程序死循环。操作系统学习的意义，就是让你从“它坏了”走向“我可以定位是哪类资源出了问题”。
