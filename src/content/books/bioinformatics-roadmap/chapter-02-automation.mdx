计算机学习的第一道门槛不是编程语言，而是文件系统。只要你不理解文件放在哪里、程序从哪里读取文件、结果保存到哪里，后面的 Python、数据分析、网页开发都会不断出错。很多报错看起来很高级，本质上只是“路径写错了”。

文件是信息的容器，目录是组织文件的方式。一个项目通常不是一个孤立文件，而是一组文件的协作：代码文件、数据文件、配置文件、说明文档、输出结果。学习时要尽早养成项目目录意识。例如一个最小项目可以这样安排：

```text
my-first-project/
  README.md
  data/
  src/
  output/
  notes/
```

`README.md` 用来说明项目目的和运行方法，`data` 放输入数据，`src` 放源代码，`output` 放结果，`notes` 放过程记录。这个结构看似简单，却能帮助你把“临时试一试”变成“别人也能理解的作品”。

路径分为绝对路径和相对路径。绝对路径从磁盘根部开始，例如 Windows 上的 `C:\Users\Name\Documents\CS-Learning`，或者 macOS/Linux 上的 `/Users/name/Documents/CS-Learning`。相对路径则相对于当前所在目录，例如 `data/input.csv`。初学者最好经常问自己：我现在在哪个目录？我要访问的文件相对于这里在哪里？

扩展名也很重要。`.txt` 是纯文本，`.md` 是 Markdown 文档，`.py` 是 Python 脚本，`.csv` 是表格数据，`.html` 是网页结构，`.json` 是结构化数据。扩展名不是装饰，它告诉系统和软件应该用什么方式理解文件。不要把 `report.docx` 改名成 `report.pdf` 就以为它变成了 PDF；真正的格式转换需要软件重新生成文件。

本章的操作练习是整理你的学习目录。创建一个 `computer-100` 文件夹，下面放 `chapter-02-paths`。在里面新建 `data/example.txt`，写入一句话，再新建 `notes/path-notes.md`，记录这个文件的绝对路径和相对路径。然后尝试把 `example.txt` 移动到别的位置，观察路径如何变化。这个练习很小，但它会让你第一次感受到：计算机不是神秘黑箱，而是一套非常严格的组织规则。

常见误区有三个。第一，把所有文件都放桌面，导致项目无法迁移。第二，文件名里混用空格、中文、特殊符号，后续命令行处理更麻烦。第三，不知道当前工作目录，复制别人的命令后直接失败。解决办法也很朴素：固定项目结构，命名尽量清晰，运行命令前先确认位置。路径理解清楚以后，编程会轻松很多。
