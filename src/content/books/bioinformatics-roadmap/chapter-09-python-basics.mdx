Python 适合作为第一门编程语言，因为它语法相对清晰，生态丰富，能覆盖自动化、数据分析、机器学习、网页后端和科研计算。学习 Python 的目标不是背完所有语法，而是尽快进入“能写小脚本解决真实问题”的状态。

最小语法集合包括变量、基本类型、列表、字典、条件、循环、函数和文件读写。变量保存数据，字符串表示文本，整数和浮点数表示数字，布尔值表示真假。列表适合保存一组有顺序的数据，字典适合保存键值关系。例如一个学生可以表示为 `{"name": "Li", "score": 92}`。条件和循环负责控制流程，函数负责复用逻辑。

一个最小 Python 脚本通常是这样的：

```python
names = ["Ada", "Linus", "Grace"]

for name in names:
    print(f"Hello, {name}")
```

这段代码展示了列表、循环和格式化字符串。不要小看它。只要把 `names` 换成文件列表、网页链接列表、实验样本列表，你就进入了自动化处理的世界。

文件读写是初学者必须掌握的能力。很多真实任务都不是在屏幕上输入一个数字，而是读取文件、处理内容、写出结果。你可以从纯文本开始，再进入 CSV、JSON 和 Excel。学习文件操作时，一定要注意路径、编码和覆盖风险。写程序前先备份数据，输出结果时尽量写到 `output` 目录，不要直接覆盖原始文件。

学习 Python 时不建议一开始沉迷刷语法题。更好的做法是围绕小任务学习：批量重命名文件，统计文本词频，清洗表格，画一张简单图，调用一个 API，整理一组文献标题。每个任务都会逼你学习一点新语法，但语法始终服务于问题。

本章练习：写一个脚本读取 `notes.txt`，统计其中有多少行、多少个非空字符，并把结果写入 `summary.txt`。如果暂时不会写代码，可以先让 AI 给你第一版，然后逐行解释每句含义，再自己修改变量名和输出格式。真正的学习发生在“运行、解释、修改、再运行”的过程中。

常见误区是复制代码后不运行，运行后不观察，报错后不读错误信息。Python 报错虽然看起来吓人，但通常会告诉你错误类型、文件名和行号。学会读 traceback，是从“靠猜”走向“会调试”的关键一步。
