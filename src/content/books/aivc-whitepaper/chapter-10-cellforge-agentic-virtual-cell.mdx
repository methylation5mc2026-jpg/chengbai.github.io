> 阅读论文：Tang X. et al. "CellForge: Agentic Design of Virtual Cell Models", arXiv, 2025/2026 version。本文选自 Canchen AI Group 的 AI Virtual Cell 文献库，并结合论文原文与 arXiv 页面整理。注意：本文为预印本阅读笔记，结论应按预印本证据等级理解。

## 编者按：从“训练一个模型”到“自动设计模型”

前面几章讨论的是单细胞基础模型本身：Geneformer 学网络语境，scGPT 学生成式细胞表征，CellFM 探索规模化，Nicheformer 引入空间语境。CellForge 则把问题推进到另一个层次：我们能不能让 AI agent 自动设计用于虚拟细胞任务的模型？

这篇论文很新，也更激进。它不是提出一个固定的细胞基础模型，而是提出一个多智能体框架。给定原始单细胞多组学数据和任务描述，CellForge 让不同专家 agent 协作分析任务、设计方法、生成可执行代码，并在扰动预测任务中产生候选神经网络架构。作者强调，核心贡献不是某个手工设计模块，而是证明多智能体协作可以生成可执行、高质量的计算生物学方法。

这对 AIVC 白皮书很重要。因为虚拟细胞的长期形态很可能不是“一个模型回答所有问题”，而是“一个自动化科研系统不断根据数据、任务、文献和实验反馈设计模型”。CellForge 展示的正是这种系统化方向的早期样子。

## 一、CellForge 要解决什么问题

虚拟细胞建模面对的任务非常复杂。不同数据集可能包含基因敲除、药物处理、细胞因子刺激；不同模态可能是 scRNA-seq、scATAC-seq、CITE-seq；不同研究问题可能要求预测平均表达、差异基因、状态转移、时间轨迹或组合扰动。很难有一个手工模型同时适配所有情况。

传统做法通常是研究者阅读文献、理解数据、选择基线模型、修改架构、写代码、跑实验、调参，再重复迭代。这个流程对跨学科能力要求很高，也非常耗时。CellForge 的设想是把其中一部分抽象成 agentic workflow：让系统先分析任务，再组织专家讨论，再生成模型和代码，再执行实验。

这不是简单 AutoML。AutoML 通常在给定搜索空间里调参或选择模型；CellForge 试图让 agent 进行更高层次的方法设计，例如提出 trajectory-aware encoder 或 perturbation diffusion module 这样的结构。也就是说，它要自动化的不只是参数搜索，而是“科研方法构思”的一部分。

## 二、三段式框架：任务分析、方法设计、实验执行

论文把 CellForge 拆成三个核心模块。

第一是 Task Analysis。系统需要读入数据和任务描述，判断数据模态、扰动类型、样本结构、潜在生物机制和适合参考的文献。这个阶段相当于一个计算生物学研究者在开题前做背景调研和数据体检。

第二是 Method Design。多个专家 agent 从不同角度提出模型设计方案，再通过讨论、评价和协调形成候选架构。这里的关键不是“一个大模型一次性回答”，而是让不同角色互相制衡：有人关注模型架构，有人关注基线，有人关注生物机制，有人关注实验可执行性。多智能体的价值，来自结构化分工和批判性协作。

第三是 Experiment Execution。系统把高层方法计划转化为可运行代码，训练模型并进行评估。如果这一环节缺失，agent 设计就容易停留在纸面方案。CellForge 强调生成 executable implementations，说明它的目标是把想法落到实验流水线里。

这个三段式结构非常像一个小型自动科研团队。AIVC 的未来如果要进入真实科研场景，可能也会采用类似结构：一个模块理解生物问题，一个模块设计模型和实验，一个模块执行计算和记录结果，一个模块把失败反馈回去。

## 三、评估任务：扰动预测是虚拟细胞的试金石

CellForge 选择在单细胞扰动预测上评估，包括基因敲除、药物处理和细胞因子刺激等场景，覆盖 scRNA-seq、scATAC-seq、CITE-seq 等模态。这一选择很合理，因为扰动预测最接近虚拟细胞的核心问题。

如果模型只能描述已有细胞状态，它只是分析工具；如果模型能预测干预后的状态，它才开始接近模拟器。基因敲除测试模型对基因调控网络的理解，药物处理测试模型对状态转移和药理效应的捕捉，细胞因子刺激则涉及信号通路和免疫相关响应。多模态数据进一步提高难度，因为模型需要在不同测量层面间建立联系。

论文声称 CellForge 在多个数据集上产生的模型具有竞争力，并展示多智能体协作相对单 agent 或人工设计的优势。这里要谨慎：作为预印本，这些结果还需要社区复现和更严格外部验证。但即便如此，它提出的问题已经值得重视：未来 AIVC 的竞争，可能不只是“谁有最大基础模型”，还包括“谁能最快把新数据和新任务转化为可靠模型”。

## 四、最值得学习的部分：把科研判断拆成可协作流程

我认为 CellForge 最有启发的地方，不是某个具体架构，而是它把科研判断拆成了流程。一个研究者面对新任务时，会做很多隐性判断：这个数据有没有批次问题？扰动类型是什么？适合用图模型、生成模型还是动力学模型？差异基因指标是否重要？模型预测应该看全基因表达还是关键通路？哪些基线必须比较？这些判断平时藏在人脑里，很难被系统化。

CellForge 尝试把这些判断显式化。Task Analysis 把数据和任务描述成报告，Method Design 让不同专家角色提出和批评方案，Experiment Execution 把方案落地。无论它当前效果多强，这种结构本身都值得学习。因为 AI for Biology 的真正壁垒，往往不在某个单独模型，而在从问题定义到实验执行的全链路判断。

对我个人来说，这正好对应项目式学习的方法：不是只读论文，而是把论文拆成问题、数据、模型、评估、局限和下一步实验。CellForge 像是把这种学习方式自动化了一部分。

## 五、AIVC 的系统形态：模型、agent 与实验闭环

如果把 CellForge 放进 AIVC 路线图，它代表的是“虚拟细胞系统层”。基础模型提供细胞和基因表征，空间模型提供组织语境，扰动模型提供干预预测，而 agent 框架负责根据具体任务组合这些能力、选择方法、生成代码和推动实验迭代。

未来的 AIVC 可能会有几层结构。

第一层是数据层，包括单细胞、空间组学、蛋白组学、扰动实验、显微图像和文献知识。

第二层是基础表征层，包括 Geneformer、scGPT、CellFM、Nicheformer 这类模型或它们的后继者。

第三层是任务模型层，针对扰动预测、细胞命运转分化、药物响应、疾病靶点排序等问题进行专门建模。

第四层是 agentic orchestration，也就是像 CellForge 这样的系统，根据任务自动调动数据、文献、模型、代码和评估。

第五层才是实验闭环：模型提出候选，实验验证，负结果回流，模型更新。

这样看，CellForge 的位置不是替代前面几章，而是把它们组织起来。它让 AIVC 从“一个模型”变成“一个会设计模型的科研流程”。

## 六、必须保留的警惕：agent 会写代码，不等于会做科学

CellForge 令人兴奋，但也最需要警惕。多智能体系统很容易制造一种自动化幻觉：报告很完整，讨论很专业，代码能运行，结果有表格，于是看起来像科学已经被自动完成。但科学的关键不只是生成结果，而是结果是否可信、假设是否可检验、错误是否能被发现。

第一，agent 可能编造不可靠的解释。即使代码可运行，方法选择背后的生物学理由也可能是薄弱的。论文自身也讨论了推理错误、质量控制和效率问题，这说明作者意识到风险。

第二，自动设计的模型可能过拟合 benchmark。扰动预测数据集数量有限，如果系统反复围绕这些任务优化，可能学到的是数据集特定套路，而不是一般虚拟细胞建模能力。

第三，自动生成代码需要严格审计。科研代码不仅要能跑，还要版本可追踪、数据不泄漏、评价无偏、随机种子和环境可复现。AIVC 不能建立在“看起来跑通”的基础上。

第四，agent 系统的成本和复杂度本身也需要评估。多智能体协作可能提高输出质量，但也引入更多 token 消耗、系统状态和调试难度。未来真正可用的平台必须证明这种复杂度带来的收益超过成本。

## 七、我的行动清单

读 CellForge 后，我会把 AIVC 学习路线补上三块。

第一，学习 perturbation prediction benchmark。理解 GEARS、scGen、CellOT、CPA 等基线大概解决什么问题，指标如何计算，数据如何划分，哪些设置容易数据泄漏。

第二，学习 agent workflow 的工程结构。包括任务规划、检索增强、专家角色设计、代码生成、自动评测和日志记录。不是为了追概念，而是为了知道一个自动科研系统怎样才可控。

第三，训练“科学审计”能力。自动化越强，越要会检查。数据来源、预处理、模型选择、评价指标、显著性、负结果、失败案例，都必须能追溯。

第四，保持预印本证据边界。CellForge 提供了很有价值的方向，但它不是已经被广泛验证的结论。写作时必须把“有前景的框架”与“已被社区确认的基础设施”分开。

## 结语：AIVC 不只是模型，而是新的科研组织方式

CellForge 最打动我的地方，是它把虚拟细胞从模型问题推向科研组织方式问题。过去我们问：能不能训练一个模型预测细胞响应？现在它让我们问：能不能让一个系统根据数据和问题自动设计这种模型？

这个问题很大，也很危险。它可能带来科研效率的巨大提升，也可能带来自动化噪声和不可审计结论。因此，真正值得追求的不是“让 AI 替代科学家”，而是让 AI 承担重复、搜索、实现和初筛工作，把人类研究者的注意力释放到问题定义、机制判断、实验设计和伦理边界上。

如果把前面几章连起来看，AIVC 的路线逐渐清晰：先学习基因网络语境，再构建生成式细胞表征，再探索规模化，再引入空间组织语境，最后让 agent 把数据、模型和实验组织成可迭代系统。CellForge 站在这条路线的系统层入口。它还不成熟，但它很值得关注，因为它提示我们：未来的 AI for Biology 竞争，不只是模型能力竞争，更是科研流程设计能力竞争。

## 再追问：自动化越强，人的判断越重要

CellForge 最适合放在这一组笔记的最后，因为它把前面的模型能力重新组织成流程能力。但流程越自动，人的责任越不能消失。研究者需要决定问题是否值得问，数据是否足够支持，模型失败是否有信息量，实验验证是否符合伦理和成本边界。未来优秀的 AI for Biology 学习者，可能不是亲手写每一行模型代码的人，而是能审计自动化系统、提出好问题、识别虚假确定性的人。
