当一个程序需要使用另一个程序的能力时，通常会通过 API 交流。API 可以理解成约定好的接口：你按照规定发送请求，对方按照规定返回结果。天气应用获取天气，网页加载评论，脚本查询数据库，AI 工具调用模型，背后都可能是 API。

Web API 常通过 HTTP 工作。你向某个 URL 发送请求，可能带上参数、请求头和请求体，服务器返回状态码和数据。返回数据最常见的格式之一是 JSON。JSON 看起来像 Python 的字典和列表组合，例如：

```json
{
  "name": "Ada",
  "skills": ["math", "programming"],
  "active": true
}
```

JSON 的优点是结构清晰，适合机器读取，也比较容易让人理解。学习 API 时，你要关注四件事：接口地址是什么，需要什么参数，返回格式是什么，错误时如何提示。很多官方 API 文档都会按这四类组织。

调用 API 时还要理解身份验证。有些接口公开可访问，有些需要 API key 或 token。密钥相当于你的身份凭证，不应该随便上传到公开仓库。初学者经常把密钥写进代码再推到 GitHub，这是非常危险的习惯。更安全的做法是使用环境变量或本地配置文件，并把配置文件加入 `.gitignore`。

API 学习的价值在于把你的程序接入更大的生态。你可以调用文献数据库接口获取论文信息，调用地图接口显示地点，调用模型接口生成文本，调用 GitHub API 查询仓库。程序不再只是本地小脚本，而可以成为连接服务的自动化工具。

本章练习：找一个无需密钥的公开 API，例如返回随机名言或公开测试数据的接口。用浏览器打开它，观察返回的 JSON。然后用 Python 的 `requests` 库请求这个地址，打印其中一个字段。最后故意访问错误路径，观察状态码和错误信息。

常见误区是只看成功示例。真实 API 调用必须考虑失败：网络断开、参数错误、权限不足、返回为空、限流、格式变化。专业程序不是假设一切顺利，而是为不顺利准备清晰的处理逻辑。
