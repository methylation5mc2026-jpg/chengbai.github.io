很多计算机教程最终都会遇到 Linux 和命令行。原因不是 Linux 更神秘，而是大量服务器、科研计算环境、开源工具和云平台都运行在类 Unix 环境中。即使你使用 Windows，也值得理解基本命令、路径和环境管理。

命令行的核心是用文本指令操作系统。常见命令包括查看当前位置、列出文件、切换目录、复制移动文件、创建目录、查看文件内容。刚开始可以只记少量高频命令，然后在真实任务中逐渐扩展。命令行不是为了替代图形界面，而是为了让重复操作可以被记录、复制和自动化。

环境是程序运行所依赖的一组条件，包括解释器版本、第三方库、系统变量和配置文件。同一个 Python 项目，在一台电脑能运行，另一台不能运行，常常是因为环境不同。包管理器负责安装和管理依赖，例如 Python 的 `pip`，Node.js 的 `npm`，conda 生态中的 `conda`。

虚拟环境解决的是项目隔离问题。不同项目可能需要不同版本的库。如果全都装到系统环境里，迟早会互相冲突。为每个项目创建虚拟环境，可以让依赖更清楚，也更容易复现。初学者要养成看 `requirements.txt`、`package.json`、`environment.yml` 的习惯，因为这些文件记录了项目需要什么。

命令行学习要特别注意当前目录和破坏性命令。查看、列出、打印通常比较安全；删除、覆盖、移动则要谨慎。运行网上复制来的命令前，至少要知道它大概做什么，作用在哪个目录。不要为了省事随便使用管理员权限。

本章练习：在一个新项目里创建 Python 虚拟环境，安装 `requests`，写一个脚本打印版本信息，然后导出依赖文件。记录完整步骤：创建环境、激活环境、安装依赖、运行脚本、退出环境。以后每做一个项目，都按类似方式记录。

常见误区是把环境问题看成玄学。其实环境问题大多可以拆成几个问题：我用的是哪个解释器，当前目录在哪里，依赖是否安装，版本是否匹配，系统能否找到命令。把这些问题逐一排查，命令行就会从恐惧来源变成可靠工具。
