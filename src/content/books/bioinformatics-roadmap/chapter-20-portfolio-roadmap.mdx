计算机学习最后要回到作品。看过多少教程、收藏多少资料、问过多少 AI，都不如一个能打开的网页、一个能运行的脚本、一份清楚的数据分析报告有说服力。作品集不是包装，而是学习过程的证据。

一个好的初学者作品不需要复杂，但要完整。完整意味着有明确问题、可运行代码、示例数据、结果展示、README 和复盘。比如“批量整理学习资料的小脚本”“个人知识库静态网站”“公开数据集的简单分析报告”“调用 API 的天气查询工具”，都可以成为早期作品。

作品集应该体现成长路径。第一阶段可以展示基础工具：命令行、Git、Python 小脚本。第二阶段展示数据能力：CSV 清洗、可视化、SQL 查询。第三阶段展示 Web 和 API：个人网站、接口调用、小应用。第四阶段展示 AI 和机器学习：一个小模型、一次评估、一个明确的错误分析。这样的路径比堆砌技术名词更可信。

复盘比结果更重要。每个项目结束后写四段：我原本想解决什么问题，我实际做出了什么，过程中最大的困难是什么，下一版可以怎样改进。这样的复盘能让别人看到你的思考，也能让你自己积累方法。优秀学习者不是从不失败，而是失败后能提炼出下一步。

公开展示要注意质量控制。不要把所有练习都放到首页最显眼位置。个人网站应该放精选项目和高质量文章，GitHub 可以保留更完整的过程。首页负责建立第一印象，仓库负责提供证据链，笔记负责记录成长。不同位置承担不同功能。

本章练习：为自己设计一个 30 天计算机学习作品计划。选择三个小项目：一个自动化脚本，一个数据分析，一个网页或 API 应用。每个项目限定一周完成，最后一周用来整理 README、写复盘、发布到 GitHub 或个人网站。不要等待“学完再做”，做项目本身就是学习。

常见误区是把作品集当简历装饰。真正有价值的作品集，应该能让读者看见你的问题意识、执行能力和迭代过程。计算机从0到100的关键，不是从零跳到满分，而是建立一种可持续升级的系统：学一点，做一点，记录一点，公开一点，再改一点。
