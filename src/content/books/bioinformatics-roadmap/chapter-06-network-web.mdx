你在浏览器里输入一个网址，按下回车，网页就出现了。这个过程看起来简单，背后却串起了计算机网络的核心概念：域名、IP、DNS、HTTP、服务器、浏览器、HTML、CSS、JavaScript。理解它们，不只是为了做网页，也会帮助你理解 API、爬虫、云服务和 GitHub Pages。

URL 是资源地址，例如 `https://example.com/articles/intro`。其中 `https` 是协议，`example.com` 是域名，后面的部分是路径。域名是给人看的名字，真正通信时需要找到服务器的 IP 地址。DNS 就像互联网的电话簿，负责把域名解析成 IP。浏览器找到服务器后，会发送 HTTP 请求，服务器返回 HTTP 响应，响应里通常包含 HTML、CSS、JavaScript、图片等资源。

HTML 描述网页结构，CSS 描述样式，JavaScript 负责交互。一个网页不是一张图片，而是一组资源被浏览器解释后的结果。你看到按钮、导航栏、文章目录和搜索框，背后都是代码、样式和数据的组合。学习 Web 的好处是反馈很快：改一行文字，刷新页面就能看到结果。

HTTP 请求有方法。最常见的是 GET 和 POST。GET 通常用于获取资源，比如打开文章页；POST 通常用于提交数据，比如登录、提交表单。响应也有状态码：`200` 表示成功，`404` 表示资源不存在，`500` 表示服务器内部错误。以后排查网站问题时，先看状态码，往往比凭感觉猜更可靠。

前端和后端是 Web 开发中的两个视角。前端负责用户在浏览器中看到和操作的部分，后端负责数据存储、身份验证、业务逻辑和接口服务。静态网站可以没有传统后端，因为它把页面提前生成好，直接托管在 GitHub Pages 这样的服务上。个人网站通常适合从静态站点开始，因为部署简单、维护成本低。

本章练习：打开任意网页，按 F12 打开开发者工具，切换到 Network 面板，刷新页面。观察浏览器请求了多少资源，每个资源的状态码是多少，哪些是 HTML，哪些是 CSS、JS、图片。不要急着理解全部细节，只要先建立一个直觉：网页是一次又一次请求和响应拼出来的。

常见误区是把“能打开网页”理解成单一动作。实际上，任何一环都可能出错：域名解析失败，网络连接失败，服务器拒绝访问，路径不存在，资源加载慢，脚本执行错误。掌握网络基础，会让你从普通用户变成能排查问题的人。
