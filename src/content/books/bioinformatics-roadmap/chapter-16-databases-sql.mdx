当数据越来越多，简单文件就不够用了。数据库的作用是可靠地存储、查询和管理结构化数据。你可以把关系型数据库理解成一组相互关联的表，每张表有行和列。行是一条记录，列是字段。学生表、课程表、成绩表、订单表、样本表，都可以用这种方式表示。

SQL 是关系型数据库的查询语言。最常见的操作包括 `SELECT` 查询数据，`WHERE` 过滤条件，`ORDER BY` 排序，`GROUP BY` 分组统计，`JOIN` 连接多张表。SQL 的好处是表达能力强，而且很多数据库都支持类似语法。学会 SQL 后，你会更理解数据产品、后台系统和科研数据库如何组织信息。

主键是表中唯一标识一条记录的字段。例如学生表可以用学号作为主键。外键表示表与表之间的关系，例如成绩表中的学号可以指向学生表。这个思想非常重要：不要把所有信息都塞进一张巨大表，而要根据实体和关系拆分。好的数据结构能减少重复和错误。

一个简单查询可以这样理解：

```sql
SELECT name, score
FROM scores
WHERE subject = 'math'
ORDER BY score DESC;
```

它的意思是：从成绩表中找出数学科目的姓名和分数，并按分数从高到低排列。SQL 接近自然语言，但执行非常严格。列名写错、表名写错、条件不清楚，结果都会出问题。

数据库学习还要注意数据修改风险。`INSERT` 添加，`UPDATE` 修改，`DELETE` 删除。查询错了通常只是结果不对，删除错了可能造成真实损失。初学阶段可以使用 SQLite 练习，因为它轻量、无需复杂服务器，适合本地小项目。

本章练习：用 SQLite 或在线 SQL 练习环境创建一张 `books` 表，包含 `id`、`title`、`author`、`year`。插入五本书，然后查询 2020 年之后出版的书，按年份排序。再增加一张 `notes` 表，用 `book_id` 关联书籍，体会多表关系。

常见误区是把数据库当成“大一点的 Excel”。数据库不仅保存数据，还提供约束、索引、事务、权限和并发控制。对初学者来说，先掌握表结构和查询语言，就已经能显著提升数据思维。
